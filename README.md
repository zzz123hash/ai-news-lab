# OmniHex Lab

OmniHex Lab is a low-cost content site for AI news, tutorials, prompts, tools, and multilingual technology briefs.

## Stack

- Astro
- TypeScript
- Markdown content collections
- RSS feed
- Sitemap
- Cloudflare Pages static deployment

The early version does not use a database or image storage. Articles are managed as Markdown files and can later be generated by an RSS or automated publishing workflow.

## Content Structure

- `src/content/blog/` stores Markdown posts.
- `src/pages/[category]/index.astro` generates category pages for `/ai/`, `/news/`, `/tutorials/`, `/prompts/`, `/tools/`, and `/rss/`.
- `src/pages/briefs/index.astro` groups `/briefs/` posts by language.
- `src/pages/[category]/[...slug].astro` generates article pages at `/{category}/{slug}/`.
- `src/pages/rss.xml.js` generates the RSS feed.
- `public/robots.txt` points crawlers to the sitemap.

Supported frontmatter:

```yaml
title: "Post title"
description: "Post description"
pubDate: 2026-05-01
category: "ai"
tags: ["AI", "Agent"]
language: "en"
source: "OmniHex Lab"
sourceUrl: "https://lab.omnihex.xyz"
draft: false
```

Supported categories:

- `ai`
- `news`
- `tutorials`
- `prompts`
- `tools`
- `briefs`
- `rss`

## Local Development

```sh
npm install
npm run dev
```

Default local URL:

```text
http://localhost:4321
```

## Build

```sh
npm run build
```

The production output is generated in `dist/`.

## Cloudflare Pages Deployment

Use these Cloudflare Pages settings:

- Framework preset: `Astro`
- Build command: `npm run build`
- Output directory: `dist`
- Node.js version: `22` or newer

Production domain:

```text
https://lab.omnihex.xyz
```

## Future Automation

A later automation flow can:

1. Pull items from RSS or source APIs.
2. Generate a Markdown draft with frontmatter.
3. Add `source` and `sourceUrl`.
4. Run editorial review.
5. Commit and deploy through Cloudflare Pages.
